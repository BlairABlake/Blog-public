Data Structures in C++

2023-09-11

#c++#programming#data-structure

Linear Data Structures

A linear data structure is a data structure where linear order is defined.

about elements

The type of elements in a container should implement:

  • copy constructor
  • copy assignment
  • move constructor
  • move assignment
  • constructor
  • destructor
  • equal operator sign (==)
  • less than operator sign (<)

Iterator

iterators are abstraction over the ways to access the linear containers. Each linear data structure provides its own iterator, but adheres to the same interface. Iterators work just like pointers.

Specifying Range with Iterator

When specifying the range with iterator by specifying start and end, the range is [start, end).

copy and move

When a value is added to a container:

lvalue is copyied.

rvalue is moved.

vector

A vector is a dynamic array.

header

#include <vector>

initializations

  • vector(n): create a fixed size vector
  • vector(n, initial value)
  • vector(elements)
  • vector(): create a dynamic size vector

iterators

  • begin
  • end: end doesn’t refer to the last element but end - 1 refer to the last element
  • rbegin
  • rend
  • cbegin
  • cend
  • crbegin
  • crend

c indicates const, and therefore it’s immutable.

r indicates reverse, and thus goes the other way around compared to the normal operator.

capacity

  • size
  • reize(capacity)

modifiers

  • push_back(element)
  • insert(iter, element)
  • pop_back()
  • erase(beg_iter, end_iter)
  • emplace(iter, constructor arguments): emplace creates a new object and inserts it
  • emplace_back(constructor arguments)
  • assign(): the arguments are the same as that of initialization

deque

A deque is basically identical to a vector, except that elemets are scattered across memory as in the linked list. However simiar to the linked list, deque allows us to access any elemtn in constant time. Also inseting an element to the beginning is more efficient than vector.

For long sequence where reallocation become expensive, deque is more efficient than vector.

header

<deque>

additional modifier to vector

  • push_front
  • pop_front

list

A list implements a doubly linked list, thus accessing to an individual element take linear time but insertion and deletion at any point can be done in constant time.

header

#include <list>

additional modifiers to vector

  • push_front
  • pop_front

operations

  • splice(iter, list): insert a given list to the position the iter refers to

  • remove(val): remove all the element of val

  • remove_if(predicate class): remove all elements for which the predicate is evaluated to be true

    A predicate class is a function or fanctor which returns a boolean value.

  • unique: remove all duplicated elements. The list must be sorted.

  • sort

  • merge: merge the sorted lists

  • reverse

array

An array is a fixed size sequence container. Unlike vector which is dynamical array, array doesn’t allocate extra memory.

header

#include <array>

queue

A queue is a container following the First-In-First-Out(FIFO) principle.

header

#include <queue>

modifiers

  • push
  • emplace
  • pop

access

  • front
  • back

priority queue

A priority queue is a queue that keeps its elemtns in sorted order. The element at the top is the one with the highest priority. The type of elements must implement less that operator sign(<).

priority queue works just like a queue in other points.

header

#include <queue>

access

  • top: return the element with the highest priority

stack

a stack is a container following the First-In-Last-Out(FIFO) principle.

stack works just like a queue in other points.

header

#include <stack>

Associative Containers

Associative containers provide a mpping of keys to values. They generally offer insertion, deletion, and lookup.

pair

pair is a utility type for storing a pair of values:

std::pair<T, U>

header

#include <utility>

initialization

std::make_pair(first, second);

access

  • first: the first element of the pair
  • second: the second element of the pair

map

map<T, U> map the element of the type T to the element of the type U. Two elements with the same key cannot be coexist.

header

#include <map>

initialization

std::map<T, U> = { pairs or initializer lists };

modifiers

  • insert(a pair or an initializer list)
  • erase(key)
  • emplace(key, arguments)

operator[]

lookup, insertion, and deletion can be done using operator[].

std::map<string, int> m = { {"A", 1}, {"B", 2} };
m["C"] = 100; //insertion
m["C"] = 3; // update
m["C"] // lookup

multimap

multimap is mapping where elements with the same key can coexist. It’s basically the same as the map but doesn’t provide operator[].

header

#include <map>

operations

  • find(k): return an iterator pointing to the first element with key equal to k.
  • count(k): return the number of elements with key equal to k
  • lower_bound(k): return an iterator pointing to the first element with key equal to k.
  • upper_bound(k): return an iterator pointing to the first elemtn whose key is considered to go after k

To iterate through the element with the same key:

auto itlow = m.lower_bound(k);
auto itup = m.upper_bound(k);
for (auto iter = itlow; iter != itup; iter++) {

}

set

set is similar to map, except that instead of storing key/value pairs, in set the value itself is the key. So the elements in a set are unique.

header

#include <set>

multiset

A multiset is a set what a multimap is to a map.

header

#include <vector>

Unordered Associative Containers

map, multimap, set, and multiset sort the elemnts with key. There are unordered counterparts to each of them, unordered_map, unordered_multimap, unordered_set, and unordered_multiset. The difference is that lookup, insertion and deletion can be done in constant time with unoredered ones whereas it take log(n) time in ordered ones.

Unordered associative containers are usually called as hash tables.

headers

#include <unordered_map> // for unordered_map, unordered_multimap
#include <unordered_set> // for unordered_set, unordered_multiset
